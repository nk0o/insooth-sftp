INSOOTH UIUX 원칙

'여정을 방해하지 않는 디자인'

모든 움직임에는 이유가 있어야 한다.
Design Rule:모든 애니메이션은 사용자의 여정을 방해하지 않고 자연스럽게 이어주는 역할만 수행
- 부드러운 Fade
- 최소한의 Motion
- 스크롤 흐름 강화
- 콘텐츠 집중도 향상

자연이 가장 먼저 보여야 한다.
Design Rule: 효과보다 사진, 장식보다 소재, 그래픽보다 실제 필드 경험을 우선

제품보다 여정을 먼저 보여준다.
Design Rule: 자연과 경험 중심의 비주얼, 제품은 경험을 완성하는 도구, 판매보다 브랜드 경험 우선

조용한 자신감을 디자인한다.
Design Rule: 과한 장식이나 효과보다 절제된 레이아웃과 균형 있는 타이포그래피로 브랜드의 자신감을 표현
- 넓은 여백
- 절제된 컬러
- 최소한의 인터랙션
- 명확한 정보 구조

모든 요소는 균형을 가져야 한다.
Design Rule: 레이아웃, 여백, 이미지, 타이포그래피, 인터랙션까지 모두 안정적인 균형을 유지합니다.
- 일관된 Grid
- 일정한 Spacing

Voice & Tone Reference

톤 방향단단함, 기능 중심, 성능 기반 — 과장 없이 담백하게
문장 구조간결 → 핵심 → 이유 순서로 구성
어휘견고함 · 안정성 · 기능을 나타내는 단어 위주, 수식어 최소화
균형좌우 균형 / 여백균형 / 텍스트 균형 / 이미지 균형
감정 강도절제된 감성 — 소리치지 않고 확신을 전달(조용한 프리미엄/NO과시)
강조UI 보다 제품이 먼저 강조
피해야 할 것과장된 수식어, 금전적 혜택 강조, 불필요한 장식(카피)

05

Naming Convention

카페24 마스터 테마 및 FORB 클래스와 충돌하지 않도록 변수는 --ap, 클래스는 ap- 접두어 + BEM을 사용합니다.

/* Block__Element--Modifier */ .ap-btn { ... } .ap-btn__icon { ... } .ap-btn--orange { ... } /* 기존 작업 규칙과 동일하게 유지 */ .ap-category-accordion{} .ap-category-accordion__column--active{} .ap-search-live{} /* JS 전역 클래스도 Ap 접두어 유지, IIFE + window.onload 패턴 */
